# Narrative Pacing Rules

## Purpose

Формальные правила подачи текста в VN-сценах. Цель: убрать «стены текста», сделать каждую секунду экранного времени интерактивной.

## Hard Rules

### R1. Максимум реплик подряд

- **≤ 3 реплик подряд** без ввода игрока (выбор, клик, проверка).
- Если нужна 4-я реплика — вставить passive check или micro-choice между 2-й и 3-й.

### R2. Одна реплика = одна мысль

- Каждая реплика несёт **одну** единицу информации.
- Нельзя: «Вокзал шумный. Люди толкаются. Часы показывают 7:15. Пахнет углём.»
- Можно: «Толпа напирает. Часы бьют семь.» → следующая реплика или выбор.

### R3. Экспозиция через действие

- Мир раскрывается через **реакции NPC и действия**, не через описания.
- Нельзя: «Фрайбург — университетский город в Бадене, основанный в...»
- Можно: Носильщик толкает тебя: «Прочь с дороги, Preußen!» → ты понимаешь, что здесь не любят чужаков.

### R4. Баланс экранного времени

- **70%** — выборы, проверки, микро-последствия (игрок ДЕЙСТВУЕТ).
- **30%** — подача, установка, реакции мира (игрок СМОТРИТ).

### R5. Провал ≠ стоп

- На провале проверки: **другая информация**, не блокировка.
- Провал даёт: худшее качество зацепки, неполную картину, ложный след.
- Путь ВСЕГДА открыт.

### R6. Формат бита

Каждый бит (20–40 сек) следует структуре:

```
ПОДАЧА: 1–2 реплики (что происходит)
  ↓
ВЫБОР: 2–3 варианта ответа (что делает игрок)
  ↓
ПРОВЕРКА: passive/active check (скрытый или явный бросок)
  ↓
МИКРО-ПОСЛЕДСТВИЕ: 1 реплика (что изменилось)
```

### R7. Зацепки, не атмосфера

- Каждое событие, каждый NPC-контакт должен сеять **зацепку** к основному расследованию.
- Чистая атмосфера без зацепки — трата экранного времени.
- Зацепка может быть: имя, число, предмет, маршрут, совпадение.

## Audit Checklist

При проверке VN-сцены:

- [ ] Нет блоков > 3 реплик подряд без ввода
- [ ] Каждая реплика = одна мысль
- [ ] Нет прямых описаний мира (всё через действие/реакцию)
- [ ] Соотношение ~70/30 (выборы/подача)
- [ ] На каждом провале — альтернативная информация, не тупик
- [ ] Каждое событие содержит минимум 1 зацепку
