# Entities

Эта директория содержит основные структуры данных, бизнес-логику и управление состоянием (преимущественно с использованием Effector.js) для различных "сущностей" в игре Grenzwanderer. Сущности представляют фундаментальные концепции, такие как персонажи, игроки, предметы, сцены и т.д.

Каждая поддиректория обычно сосредоточена на конкретной сущности и может включать:

-   `model.ts` (или `model/*.ts`): Сто́ры (хранилища), события и доменная логика Effector для управления состоянием сущности.
-   `types.ts` (или встроены в `model.ts`): Интерфейсы и типы TypeScript, определяющие структуру сущности и связанных данных.
-   `api/`: Хуки React и другие функции API для работы с сущностью.
-   `ui/`: Компоненты React, связанные с отображением сущности или ее частей.
-   `lib/`: Вспомогательные функции и утилиты, специфичные для сущности.
-   `index.ts`: Публичный API модуля сущности, реэкспортирующий необходимые сто́ры, события, типы и компоненты.

## Основные сущности

### `character`
Управляет отображением и состоянием персонажей визуальной новеллы.
-   **`model/useCharacterState.ts`**: Хук React для CSS-классов позиции, активности и анимации персонажа.
-   **`ui/CharacterImage.tsx`**: Компонент для отображения спрайтов персонажей, обработки загрузки и активных состояний.
-   **`ui/CharacterName.tsx`**: Компонент для отображения имен персонажей со специфическим стилем.

### `markers`
Обрабатывает маркеры на карте, их состояния и взаимодействия для QR-квестов и точек интереса.
-   **`model.ts`**: Сто́ры Effector для данных маркеров (`$markers`, `$activeMarkers`), их видимости, статуса выполнения и взаимодействий (`$markerInteractions`). Включает начальные данные маркеров и события, такие как `showMarker`, `addMarker`, `toggleMarkerComplete`.
-   **`index.ts`**: Экспортирует сто́ры маркеров, события и соответствующие типы (`MarkerData`, `QuestMarker`).

### `player`
Управляет данными, специфичными для игрока, разделенными на различные аспекты:
-   **`model.ts`**: Обрабатывает статистику игрока в стиле визуальной новеллы (`$playerStats` для энергии, денег, социальных атрибутов, таких как привлекательность, сила воли), недавние изменения статистики (`$recentStatChanges`) и перки (`$playerPerks`).
-   **`model/playerAttributes.ts`**: Определяет и управляет основными RPG-атрибутами (`$primaryAttributes`, такими как сила, ловкость, интеллект), производными вторичными характеристиками (`$secondaryStats`, такими как здоровье, мана, сопротивления, шанс критического удара), уровнем игрока (`$playerLevel`) и опытом. Включает формулы для расчета вторичных характеристик из первичных и логику повышения уровня.
-   **`model/playerSkills.ts`**: Реализует детализированную систему навыков. Управляет путями развития (`ClassPath`), отдельными узлами навыков (`SkillNode`) со стоимостью и эффектами, опытом для каждого пути, разблокировкой навыков (`$playerSkills`) и очками навыков. Включает исчерпывающий список примеров навыков (`ALL_SKILL_NODES`).
-   **`api/usePlayer.ts`**: Хук для получения и управления данными игрока через Convex API. Обеспечивает инициализацию данных игрока, отслеживание геолокации, обновление информации о местоположении. Включает механизм отката к мок-данным игрока при недоступности API или отсутствии userId.

### `scene`
Управляет состоянием сцен визуальной новеллы.
-   **`model.ts`**: Сто́ры Effector для текущей сцены (`$currentScene`), истории диалогов (`$dialogHistory`), состояния загрузки сцены (`$sceneLoading`), эффектов перехода (`$transitionEffect`), звуковых эффектов (`$currentSound`) и скорости отображения текста (`$textSpeed`). События, такие как `setCurrentScene` и `dialogueLineDisplayed`, управляют изменениями состояния.

### `shops`
Определяет структуру и примеры данных для внутриигровых магазинов и торговли.
-   **`model/shops.ts`**: Содержит интерфейсы TypeScript для `Shop`, `ShopItem`, `ShopkeeperType`, `TradeTransaction` и т.д. Предоставляет обширные данные `EXAMPLE_SHOPS` и вспомогательные функции для расчета цен (`calculateItemPrice`), доступности товаров (`isItemAvailableForPlayer`) и генерации диалогов. Этот модуль в основном определяет структуры данных, а не активное управление состоянием.

### `user`
Управляет базовыми данными аутентифицированного пользователя.
-   **`model.ts`**: Сто́р Effector (`$currentUser`) и событие (`setUser`) для информации о текущем вошедшем пользователе (ID, email).

## Соглашения

-   **Effector.js**: Используется для управления состоянием. Сто́ры имеют префикс `$` (например, `$playerStats`).
-   **TypeScript**: Весь код написан на TypeScript со строгой типизацией структур данных.
-   **Модульность**: Каждая сущность спроектирована так, чтобы быть максимально автономной, предоставляя четкий публичный API через свой `index.ts`.
-   **Общие типы**: Общие типы могут находиться в директории более высокого уровня `shared/types` и реэкспортироваться или использоваться сущностями.

```
  │   ├── model/      # Модели и типы предметов

  ├── player/         # Сущность игрока
  │   ├── api/        # API для взаимодействия с игроком
  │   │   └── usePlayer.ts
  │   ├── model/      # Модели игрока 
  │   │   ├── playerAttributes.ts
  │   │   └── playerSkills.ts
  │   ├── model.ts    # Основная модель игрока
  │   └── index.ts    # Публичное API модуля

  └── user/           # Сущность пользователя
```

## Принципы организации

1. **Изоляция**: Каждая сущность полностью инкапсулирует свою бизнес-логику
2. **Публичное API**: Каждая сущность имеет четко определенное публичное API через index.ts
3. **Независимость**: Entities не должны зависеть от features или других слоев более высокого уровня

## Рекомендации по использованию

### Правильно

```typescript
// Импорт через публичное API
import { MarkerType, showMarker } from '../../entities/markers';
```

### Неправильно

```typescript
// Импорт напрямую из модели, минуя публичное API
import { MarkerType } from '../../entities/markers/model';
```

## Типы и константы

Определения перечислений, интерфейсов и констант выносятся в shared/types и shared/constants для предотвращения циклических зависимостей:

```typescript
// src/shared/types/marker.types.ts
export enum MarkerType { ... }
export interface MarkerData { ... }

// src/entities/markers/model.ts
import { MarkerType, MarkerData } from '../../shared/types/marker.types';
```

## Состояние

Для управления состоянием используется библиотека Effector:

```typescript
// src/entities/markers/model.ts
import { createStore, createEvent } from 'effector';
import { MarkerData } from '../../shared/types/marker.types';

// События
export const showMarker = createEvent<string>();
export const hideMarker = createEvent<string>();

// Хранилище
export const $markers = createStore<MarkerData[]>([])
  .on(showMarker, (state, id) => /* обработчик */)
  .on(hideMarker, (state, id) => /* обработчик */);
```
