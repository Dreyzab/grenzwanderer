# Директория Pages

В этой директории содержатся компоненты-страницы нашего приложения. Каждая страница представляет собой точку входа для конкретного маршрута.

## Текущие проблемы

1. **Дублирование компонентов страниц**: Обнаружены дублирующие компоненты для одной и той же функциональности.
   - `QuestLogPage` и `QuestJournalPage`
   - `CharacterStatsPage` и `CharacterPage`
   - Возможно есть и другие дублирующие компоненты

2. **Непоследовательность структуры директорий**:
   - Некоторые страницы используют структуру `FeaturePage/ui/FeaturePage.tsx`
   - Другие используют плоскую структуру `feature/FeaturePage.tsx`

3. **Несогласованный подход к экспорту компонентов**:
   - Смешаны default и named exports
   - Разные подходы к экспорту в index.ts

4. **Разное использование PageLayout**:
   - Разные способы использования компонента PageLayout
   - Непоследовательная передача props

5. **Смешанные подходы к стилизации**:
   - CSS модули (styles)
   - Tailwind CSS классы

## План рефакторинга

1. **Стандартизация структуры директорий**:
   ```
   pages/
     ├── SomePage/
     │    ├── ui/
     │    │    └── SomePage.tsx
     │    ├── model/ (если есть локальная логика)
     │    └── index.ts
     └── index.ts
   ```

2. **Единый подход к экспорту**:
   - Использовать named exports в каждом компоненте
   - В index.ts каждой директории re-export компонентов
   - В корневом index.ts публичный API для всех страниц

3. **Стандартизация использования PageLayout**:
   - Использовать компонент как обертку с передачей header и content в props
   - Пример:
     ```tsx
     <PageLayout
       header={<Header />}
       content={<Content />}
     />
     ```

4. **Единый подход к стилизации**:
   - Использовать Tailwind CSS для всех компонентов
   - Убрать CSS модули

5. **Устранение дублирования**:
   - Выбрать один набор компонентов для каждой функциональности
   - Удалить устаревшие компоненты
   - Обновить импорты в других файлах

## Порядок работы

1. Провести аудит всех страниц и определить какие компоненты следует оставить
2. Рефакторинг выбранных компонентов к новому стандарту
3. Проверить все импорты во всем проекте
4. Удалить устаревшие файлы
5. Обновить маршрутизацию
