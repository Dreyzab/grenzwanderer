# Shared - Общий слой

Shared слой содержит общие утилиты, константы, типы, UI-компоненты и сервисы, которые не привязаны к конкретной бизнес-логике и могут быть использованы в любом месте приложения.

## Структура

```
shared/
  ├── api/         # Утилиты для работы с API
  ├── assets/      # Общие ассеты (изображения, иконки)
  ├── config/      # Конфигурационные константы
  ├── constants/   # Константы
  │   ├── marker.ts      # Константы для маркеров
  │   └── quest.ts       # Константы для квестов
  ├── hooks/       # Общие хуки
  ├── lib/         # Библиотеки-обертки над сторонними решениями
  ├── services/    # Сервисы для работы с внешними источниками данных
  │   └── api.service.ts # Сервисный слой для работы с Convex API
  ├── types/       # Типы и интерфейсы
  │   ├── gameScreen.ts     # Типы для игрового экрана
  │   ├── marker.types.ts   # Типы для маркеров
  │   ├── shelter.ts        # Типы для убежища
  │   └── visualNovel.ts    # Типы для визуальной новеллы
  └── ui/          # Общие UI-компоненты
```

## Основные принципы

### 1. Независимость

Shared слой не должен иметь зависимостей от entities, features или widgets. Это самый низкоуровневый слой.

### 2. Разделение типов и реализации

- В shared/types определяются только типы и интерфейсы
- В shared/constants определяются только константы
- Реализация функциональности выносится в entities или features

### 3. Централизация переиспользуемого кода

Код, который может быть использован в разных модулях, должен быть размещен в shared.

## Использование сервисного слоя

Сервисный слой предоставляет единую точку доступа к внешним API:

```typescript
// Пример использования
import { QuestService } from '../../shared/services/api.service';

const MyComponent = () => {
  // Хук для активации квеста
  const activateQuest = QuestService.useActivateQuestByQR();
  
  const handleScan = (qrCode: string) => {
    activateQuest({ playerId, qrCode });
  };
  
  // ...
};
```

## Организация типов

Типы в shared/types организованы по доменным областям, а не по техническим категориям:

```
shared/types/
  ├── marker.types.ts   # Все типы, относящиеся к маркерам
  ├── quest.types.ts    # Все типы, относящиеся к квестам
  ├── player.types.ts   # Все типы, относящиеся к игроку
  └── ...
```

## Рекомендации по именованию

- Файлы с типами именуются с суффиксом `.types.ts`
- Файлы с константами именуются с названием домена
- Сервисы именуются с суффиксом `.service.ts`
